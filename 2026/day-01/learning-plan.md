# 90-Day DevOps & Cloud Learning Plan

## Profile & Objective
* **Current Level:** Application Support Specialist (Linux Administration, WAS, Databases)
* **Current CTC / Target:** 14.7 LPA | Targeting high-impact DevOps/SRE roles (20+ LPA)
* **Objective:** Modernize traditional application support and middleware expertise (WebLogic/JBoss/Tomcat) into cloud-native automation, Infrastructure as Code, CI/CD, and container orchestration.

---

## 1. Core Understanding & Motivation
* **What is DevOps & Cloud to me?**
  DevOps is transitioning from manual, reactive application support, server installations, and troubleshooting to proactive, automated, self-healing, and code-driven delivery pipelines. Cloud engineering is the scalable playground where this infrastructure is managed as code.
* **Why start now?**
  To leverage my deep background in Linux, middleware (Tomcat, JBoss, WebLogic), and database management to build modern cloud architectures. I want to shift from manual operations to designing and automated deployments.
* **Where do I want to reach?**
  Become a Senior DevOps/SRE Engineer capable of architecting and automating highly available, secure enterprise infrastructures on AWS/Azure using Kubernetes and Terraform.

---

## 2. 90-Day Goals & Core Skills
### Three Clear Goals:
1. **Containerize & Migrate WAS Workloads:** Package Java-based middleware applications (JBoss/Tomcat) into Docker containers and orchestrate them on Kubernetes (EKS).
2. **Infrastructure as Code (IaC):** Automate the provisioning of multi-tier cloud environments (VPC, EC2, RDS PostgreSQL/MSSQL) on AWS using Terraform.
3. **End-to-End GitOps / CI/CD:** Establish automated pipelines (GitHub Actions/Jenkins) that build, test, scan, and deploy applications to Kubernetes.

### Three Core Skills to Build:
1. **Containerization & Orchestration:** Transitioning from manual app deployments (WebLogic/JBoss) to Docker and Kubernetes (Pod lifecycle, Networking, Storage, Helm).
2. **Infrastructure as Code (Terraform):** Declarative scripting to deploy resources instead of manual server installations.
3. **CI/CD & Observability:** Automating code promotion pipelines and setting up centralized logging/monitoring (Prometheus/Grafana) to replace manual troubleshooting.

---
